# Resource-Based View (мһҗмӣҗкё°л°ҳкҙҖм җ)

> "Firms are bundles of productive resources... the heterogeneity of resources across firms explains performance differences."
> вҖ” Edith Penrose (1959), Birger Wernerfelt (1984), Jay Barney (1991)

## 1. Core Premise (н•өмӢ¬ м „м ң)

Firms are **bundles of resources and capabilities**. Performance differences arise because:
- Resources are **heterogeneously** distributed across firms (мһҗмӣҗ мқҙм§Ҳм„ұ)
- Resources are **imperfectly mobile** вҖ” they cannot be freely traded (л¶Ҳмҷ„м „ мқҙлҸҷм„ұ)
- Competitive advantage comes from **inside-out**, not just outside-in

This contrasts with Porter's industry-structure view: RBV says **what you have** matters more than **where you compete**.

**Key insight**: Tangible resources are easiest to copy. **Intangible + Human + Organizational** resources are where sustainable advantage lives.

## 3. VRIO Framework (VRIO н”„л Ҳмһ„мӣҢнҒ¬)

The VRIO test determines whether a resource can generate **sustained competitive advantage** (м§ҖмҶҚм Ғ кІҪмҹҒмҡ°мң„).

### 3.1 Four Questions

| Test | Question | н•өмӢ¬ м§Ҳл¬ё |
|------|----------|-----------|
| **V** вҖ” Valuable (к°Җм№ҳ мһҲлҠ”) | Does it exploit an opportunity or neutralize a threat? | мқҙ мһҗмӣҗмқҙ кё°нҡҢлҘј нҷңмҡ©н•ҳкұ°лӮҳ мң„нҳ‘мқ„ л¬ҙл Ҙнҷ”н•ҳлҠ”к°Җ? |
| **R** вҖ” Rare (нқ¬мҶҢн•ң) | Do few competitors possess it? | мҶҢмҲҳмқҳ кІҪмҹҒмһҗл§Ң ліҙмң н•ҳкі  мһҲлҠ”к°Җ? |
| **I** вҖ” Inimitable (лӘЁл°© л¶Ҳк°Җн•ң) | Is it costly to imitate? | лӘЁл°©н•ҳлҠ” лҚ° л№„мҡ©мқҙ л§Һмқҙ л“ңлҠ”к°Җ? |
| **O** вҖ” Organized (мЎ°м§Ғнҷ”лҗң) | Is the firm organized to capture value? | мқҙ мһҗмӣҗмқҳ к°Җм№ҳлҘј нҸ¬м°©н•ҳлҸ„лЎқ мЎ°м§Ғлҗҳм–ҙ мһҲлҠ”к°Җ? |

### 3.2 Three Reasons for Inimitability (лӘЁл°© л¶Ҳк°Җмқҳ 3к°Җм§Җ мӣҗмқё)

1. **Unique Historical Conditions (лҸ…нҠ№н•ң м—ӯмӮ¬м Ғ мЎ°кұҙ)**: Path-dependent accumulation вҖ” "you had to be there"
2. **Causal Ambiguity (мқёкіјм Ғ лӘЁнҳём„ұ)**: Competitors cannot identify which resources cause advantage
3. **Social Complexity (мӮ¬нҡҢм Ғ ліөмһЎм„ұ)**: Culture, trust, relationships cannot be engineered

## 4. Dynamic Capabilities (лҸҷм Ғ м—ӯлҹү) вҖ” Teece (1997, 2007)

When environments change rapidly, **static resources decay**. Dynamic capabilities extend RBV:

| Capability | Description | м„ӨлӘ… |
|------------|-------------|------|
| **Sensing (к°җм§Җ)** | Scan, learn, interpret вҖ” identify opportunities and threats | нҷҳкІҪ ліҖнҷ”лҘј к°җм§Җн•ҳкі  кё°нҡҢлҘј нғҗмғү |
| **Seizing (нҸ¬м°©)** | Design business models, commit resources, mobilize | кё°нҡҢлҘј нҸ¬м°©н•ҳм—¬ мһҗмӣҗмқ„ нҲ¬мһ… |
| **Transforming (м „нҷҳ)** | Reconfigure assets, realign structure, renew routines | мһҗмӮ°мқ„ мһ¬л°°м№ҳн•ҳкі  мЎ°м§Ғмқ„ ліҖнҷҳ |

**When RBV alone isn't enough**: In fast-moving industries (tech, platform), static resources lose value quickly. Dynamic capabilities = the ability to **build, integrate, and reconfigure** resources over time.

## 5. Core Competence Integration (н•өмӢ¬м—ӯлҹү вҖ” Prahalad & Hamel, 1990)

Core competence is the **collective learning** of the organization. Three tests:

| Test | Question | н•өмӢ¬ м§Ҳл¬ё |
|------|----------|-----------|
| **Customer Benefit (кі к°қ к°Җм№ҳ)** | Does it significantly contribute to perceived customer benefits? | кі к°қмқҙ мқёмӢқн•ҳлҠ” к°Җм№ҳм—җ нҒ¬кІҢ кё°м—¬н•ҳлҠ”к°Җ? |
| **Competitor Differentiation (кІҪмҹҒ м°Ёлі„нҷ”)** | Is it difficult for competitors to imitate? | кІҪмҹҒмһҗк°Җ лӘЁл°©н•ҳкё° м–ҙл Өмҡҙк°Җ? |
| **Extendability (нҷ•мһҘ к°ҖлҠҘм„ұ)** | Can it be leveraged across markets and products? | м—¬лҹ¬ мӢңмһҘкіј м ңн’ҲмңјлЎң нҷ•мһҘ к°ҖлҠҘн•ңк°Җ? |

**Relationship to VRIO**: Core competence = a resource that passes all four VRIO tests AND can be applied across multiple business lines.

## 6. Resource Audit Process (мһҗмӣҗ к°җмӮ¬ н”„лЎңм„ёмҠӨ)

### Step-by-step

```
Step 1: IDENTIFY (мӢқлі„)
  в””в”Җ List all resources the firm possesses or controls

Step 2: CLASSIFY (л¶„лҘҳ)
  в””в”Җ Categorize as Tangible / Intangible / Human / Organizational

Step 3: VRIO TEST (VRIO н…ҢмҠӨнҠё)
  в””в”Җ Apply V вҶ’ R вҶ’ I вҶ’ O sequentially to each resource

Step 4: STRATEGIC IMPLICATIONS (м „лһөм Ғ мӢңмӮ¬м җ)
  в”ңв”Җ Protect resources with sustained advantage (ліҙнҳё)
  в”ңв”Җ Invest in resources with temporary advantage вҶ’ build inimitability (нҲ¬мһҗ)
  в”ңв”Җ Monitor parity resources вҶ’ don't overspend (мң м§Җ)
  в””в”Җ Divest or ignore disadvantage resources (м •лҰ¬)
```

### When to Use Which

- **Use RBV** when: deciding what to build/acquire, evaluating diversification, assessing "can we compete?"
- **Use Porter** when: choosing which market to enter, analyzing competitive dynamics, pricing strategy
- **Integrate both** for complete strategy: Porter tells you WHERE to play, RBV tells you HOW to win

## 8. Output Format

Analysis output uses the following structure with emoji headers:

```
рҹ”Қ мһҗмӣҗ к°җмӮ¬ (Resource Audit)
  - Identified resources by category

рҹ“Ҡ VRIO л¶„м„қ (VRIO Analysis)
  - VRIO table for each resource
  - Competitive implication per resource

рҹҸҶ н•өмӢ¬м—ӯлҹү (Core Competencies)
  - Resources passing all VRIO tests
  - Core competence 3-test evaluation

вҡЎ лҸҷм Ғ м—ӯлҹү (Dynamic Capabilities)
  - Sensing / Seizing / Transforming assessment
  - Capability gaps and development priorities

рҹҺҜ м „лһөм Ғ мӢңмӮ¬м җ (Strategic Implications)
  - Protect, invest, maintain, divest recommendations
  - Resource development roadmap

рҹ”— мҷёл¶Җ м—°кі„ (External Integration)
  - Link to Porter's Five Forces / Value Chain
  - Industry context for resource value
```

## 9. Sub-Commands

### `/think:rbv` вҖ” Full Resource Analysis (м „мІҙ мһҗмӣҗ л¶„м„қ)
Complete resource-based analysis: identify вҶ’ classify вҶ’ VRIO вҶ’ core competence вҶ’ dynamic capabilities вҶ’ strategic implications.

### `/rbv:vrio` вҖ” VRIO Assessment (VRIO нҸүк°Җ)
Focused VRIO analysis of specific resources. Input: list of resources or capabilities. Output: VRIO table with competitive implications.

### `/rbv:capability` вҖ” Dynamic Capabilities Audit (лҸҷм Ғ м—ӯлҹү к°җмӮ¬)
Assess the firm's sensing, seizing, and transforming capabilities. Identify capability gaps and prioritize development. Evaluate readiness for environmental change.
