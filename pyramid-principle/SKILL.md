# Pyramid Principle (피라미드 원칙)

Barbara Minto가 McKinsey에서 개발한 **구조적 사고 및 커뮤니케이션 프레임워크**.
핵심: **결론을 먼저, 근거는 그 아래에 (Top-Down).**

---

## 1. Core Rule: Answer First (결론 선행)

**절대 규칙**: 항상 답(결론/권고)을 먼저 제시하고, 그 다음에 근거를 배치한다.

- 독자/청중은 "So what?"을 먼저 알고 싶어 한다
- 근거부터 나열하면 핵심이 묻힌다
- 경영진은 결론을 먼저 듣고, 필요하면 근거를 파고든다

```
❌ 나쁜 예: "A 조사를 했고, B 분석을 했고, C 결과가 나왔고... 그래서 X를 해야 합니다"
✅ 좋은 예: "X를 해야 합니다. 이유는 세 가지입니다: ①... ②... ③..."
```

## 2. Pyramid Structure (피라미드 구조)

```
                    ┌─────────────────────┐
                    │   Governing Thought  │  ← 핵심 메시지 (결론/권고)
                    │   (지배적 사고)        │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
        ┌─────┴─────┐   ┌─────┴─────┐   ┌─────┴─────┐
        │ Key Line 1 │   │ Key Line 2 │   │ Key Line 3 │  ← 핵심 근거 (3~5개)
        │ (핵심근거1) │   │ (핵심근거2) │   │ (핵심근거3) │
        └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
              │                │                │
         ┌────┼────┐     ┌────┼────┐     ┌────┼────┐
         │    │    │     │    │    │     │    │    │
        [데이터/사실]   [데이터/사실]   [데이터/사실]     ← 뒷받침 데이터
```

**규칙**:
- 각 레벨은 아래 레벨의 **요약(summary)** 이어야 한다
- 같은 레벨의 항목은 **동일한 종류의 아이디어**여야 한다
- 그룹 내 항목은 **논리적 순서**로 배열한다

## 3. MECE Principle (상호배제 완전포괄)

**Mutually Exclusive, Collectively Exhaustive**

| 원칙 | 의미 | 테스트 |
|------|------|--------|
| **ME** (상호배제) | 항목 간 겹침 없음 | "이 두 항목에 동시에 속하는 것이 있는가?" → No |
| **CE** (완전포괄) | 빠짐 없이 전체를 커버 | "이 항목들 외에 빠진 것이 있는가?" → No |

**MECE 그룹화 방법**:
- 프레임워크 활용: 3C, 4P, SWOT, 내부/외부, 단기/장기
- 프로세스 단계: 조달→생산→출하→A/S
- 세그먼트 분류: 지역별, 고객군별, 제품별

```
❌ Non-MECE: "영업 문제, 마케팅 문제, 고객 문제" (겹침 존재)
✅ MECE:     "수요 측 문제, 공급 측 문제, 구조적 문제" (겹침 없고 전체 커버)
```

## 4. Two Ways to Build a Pyramid (피라미드 구축 방법)

### 4a. Top-Down (하향식) — 결론을 아는 경우

1. **질문 정의**: 독자가 가진 핵심 질문은?
2. **답변 작성**: 그 질문에 대한 답(결론)을 한 문장으로
3. **근거 도출**: 답을 뒷받침하는 3~5개 핵심 근거
4. **데이터 배치**: 각 근거를 뒷받침하는 사실/수치

### 4b. Bottom-Up (상향식) — 결론이 불명확한 경우

1. **데이터 나열**: 관련 사실, 발견, 아이디어를 모두 적는다
2. **그룹화**: 유사한 것끼리 MECE하게 묶는다
3. **요약 추출**: 각 그룹의 공통 메시지를 한 문장으로 뽑는다
4. **핵심 메시지 도출**: 그룹 요약들을 아우르는 Governing Thought 작성

**실무 팁**: 대부분의 경우 **Top-Down을 먼저 시도**하고, 막히면 Bottom-Up으로 전환한다.

## 5. Logical Ordering (논리적 배열)

같은 레벨의 항목은 반드시 **하나의 논리적 순서**를 따라야 한다:

| 순서 유형 | 설명 | 예시 |
|-----------|------|------|
| **Time Order** (시간순) | 1단계→2단계→3단계 | 프로세스, 계획, 역사 |
| **Structural Order** (구조순) | 전체를 부분으로 나눔 | 지역별, 부서별, 제품별 |
| **Degree/Ranking Order** (중요도순) | 가장 중요한 것부터 | 매출 기여도, 리스크 크기 |

**검증**: "이 순서를 바꾸면 논리가 깨지는가?" → 깨지면 올바른 순서.

## 6. SCQ Framework (상황-전개-질문)

도입부(Introduction)를 구성하는 프레임워크. 독자를 "왜 이 문서를 읽어야 하는가"로 이끈다.

| 요소 | 영문 | 역할 | 예시 |
|------|------|------|------|
| **S** | Situation (상황) | 독자가 동의하는 현재 상태 | "우리 회사 연매출은 108억이며 성장 중이다" |
| **C** | Complication (전개) | 긴장/변화/문제 | "하지만 핵심 고객 1곳이 매출의 30%를 차지하여 리스크가 크다" |
| **Q** | Question (질문) | S+C에서 자연스럽게 나오는 핵심 질문 | "고객 집중도를 어떻게 낮출 것인가?" |
| → **A** | Answer (답변) | = Governing Thought | "3개년 신규 고객 다변화 전략을 실행해야 한다" |

**SCQ가 강력한 이유**: 독자의 머릿속에 질문을 심어놓고, 바로 답을 제시하므로 **설득력 극대화**.

## 7. Inductive vs Deductive Reasoning (귀납 vs 연역)

피라미드 내 그룹은 두 가지 논증 방식 중 하나를 사용한다:

### 연역적 (Deductive)
```
대전제: A이면 B이다
소전제: C는 A이다
결론:   따라서 C는 B이다
```
- 직선적, 한 단계씩 따라가야 함
- 복잡한 주제에서는 따라가기 어려움
- **Minto 권고: 가급적 피라미드에서는 귀납을 선호**

### 귀납적 (Inductive)
```
사실 1: 매출이 하락 중이다
사실 2: 경쟁사가 진입했다
사실 3: 핵심 인력이 이탈했다
→ 종합: 사업 경쟁력이 약화되고 있다
```
- 같은 종류의 사실을 모아 하나의 결론 도출
- 독자가 한눈에 파악 가능
- **피라미드의 기본 논증 방식**

## 9. Storyboarding (스토리보드)

피라미드를 프레젠테이션 슬라이드로 전환하는 방법:

### 전환 규칙
1. **Governing Thought** → 타이틀 슬라이드 또는 Executive Summary
2. **각 Key Line** → 섹션 구분 슬라이드 (divider)
3. **각 Supporting Point** → 개별 슬라이드
4. **데이터/차트** → 해당 슬라이드의 증거

### 슬라이드 원칙
- **One Slide, One Message**: 슬라이드당 하나의 메시지만
- **Action Title**: 슬라이드 제목은 "매출 현황"(X) → "매출이 20% 하락했다"(O)
- **So What Test**: 모든 슬라이드에 "So what?"을 물었을 때 답이 있어야 함

```
[슬라이드 1] 고객 다변화 전략이 필요하다 (Governing Thought)
[슬라이드 2] ── 섹션: 현재 고객 집중도가 위험 수준이다 (Key Line 1)
[슬라이드 3]    Top 1 고객이 매출 30% 차지 (데이터)
[슬라이드 4]    동종업계 평균은 15% 미만 (벤치마크)
[슬라이드 5] ── 섹션: 신규 시장 진출 여건이 갖추어졌다 (Key Line 2)
  ...
```

## 10. Output Format (출력 형식)

피라미드 원칙으로 구조화된 결과물은 다음 형식을 따른다:

```
🎯 핵심 메시지 (Governing Thought)
한 문장으로 결론/권고를 제시

📌 근거 1: [Key Line 제목]
- 뒷받침 데이터/사실
- 뒷받침 데이터/사실

📌 근거 2: [Key Line 제목]
- 뒷받침 데이터/사실
- 뒷받침 데이터/사실

📌 근거 3: [Key Line 제목]
- 뒷받침 데이터/사실
- 뒷받침 데이터/사실

⚡ 다음 단계 (Next Steps)
- 구체적 액션 아이템
```

## 11. Sub-Commands (하위 명령)

### `/think:pyramid-principle` — Structure Any Communication (커뮤니케이션 구조화)

**입력**: 주제, 분석 결과, 또는 정리되지 않은 생각들
**프로세스**:
1. 핵심 질문(Question) 파악
2. Governing Thought 도출
3. MECE 기반 Key Line 3~5개 구성
4. 논리적 순서 배열 (시간/구조/중요도)
5. 피라미드 구조 출력

**출력**: 위 섹션 10의 형식으로 구조화된 커뮤니케이션

### `/pyramid-principle:scq` — Craft SCQ Introduction (도입부 작성)

**입력**: 배경 상황, 문제, 또는 보고서 주제
**프로세스**:
1. **Situation** 작성 — 독자가 동의할 객관적 현재 상태
2. **Complication** 작성 — 긴장, 문제, 변화 포인트
3. **Question** 도출 — S+C에서 자연스럽게 나오는 질문
4. **Answer** 제시 — Governing Thought

**출력**:
```
📋 SCQ 도입부

🔵 Situation (상황):
[독자가 동의하는 현재 상태]

🔴 Complication (전개):
[문제/변화/긴장]

❓ Question (질문):
[핵심 질문]

🎯 Answer (답변):
[Governing Thought — 결론/권고]
```

### `/pyramid-principle:storyboard` — Turn Analysis into Presentation (프레젠테이션 구조)

**입력**: 분석 결과, 보고서, 또는 피라미드 구조
**프로세스**:
1. 피라미드 구조 확인/구축
2. 슬라이드 단위 분해 (1 slide = 1 message)
3. Action Title 작성 (서술형 제목)
4. 각 슬라이드의 핵심 비주얼/차트 제안
5. 스토리 흐름 검증 (So What Test)

**출력**:
```
🎬 Storyboard

[Slide 1] 🎯 [Action Title — Governing Thought]
  비주얼: [제안]

[Slide 2] 📌 섹션: [Key Line 1 Action Title]
  비주얼: [제안]

[Slide 3] 📊 [Supporting Point Action Title]
  비주얼: [차트/데이터 유형]
  데이터: [핵심 수치]

...
```
